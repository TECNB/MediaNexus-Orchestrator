# 接入用户动作每日额度

## 背景

认证第一批已提供用户、角色和 Bearer token。第二批先实现普通用户的基础使用限制，不引入管理员审核、禁用用户、额度后台或更细的集数限制。

## 范围

- 新增 `user_action_usage` 表记录用户每日动作使用次数。
- 新增用户动作类型：
  - `MAGNET_INGEST_CREATE`
  - `ANIME_SUBSCRIBE_CREATE`
- 普通用户每日两类动作合计最多 3 次。
- 管理员不受额度限制。
- 超限返回 HTTP 429 和清晰业务错误信息。
- 动漫手动磁力创建任务接入额度消费。
- 动漫订阅创建接入额度消费。

## 非目标

- 不做禁用用户。
- 不做管理员审核流。
- 不做额度后台。
- 不做前端额度展示。
- 不做手动磁力集数上限等细粒度限制。

## 验收

- 未登录调用受限创建接口返回 401。
- 普通用户当天两类创建动作累计达到 3 次后再次调用返回 429。
- 管理员调用不消耗额度、不受次数限制。
- 额度记录写入 `user_action_usage`。
- 现有登录注册和业务创建接口编译通过。
